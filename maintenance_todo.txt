--- BUG: Moderator chat stops responding ---
Sometimes in moderator chat mode the AI stops responding entirely — no error
message, just silence. Root cause unknown. Needs investigation: check whether
the Edge Function is timing out, whether the chatHistory array is growing too
large and hitting max_tokens, or whether the JSON parse is silently failing.
Look at chatWithModerator() in src/utils/claude.js and the corresponding edge
function. Add better error surfacing so the user sees something actionable
instead of nothing.

--- FEATURE: Retry logic for API overloaded errors ---
Anthropic occasionally returns HTTP 529 (overloaded) during high-traffic
periods. The current code has no retry logic, so users see a raw error.
Fix: in the Edge Function (or in src/utils/claude.js), add exponential backoff
retries — 3 attempts with 1s, 2s, 4s delays. Show the user a "retrying..."
indicator rather than an immediate error. For sustained rate limit issues,
request a limit increase from Anthropic once the app has real usage history.

--- Set up custom SMTP before launch ---
Supabase's built-in email service has rate limits and isn't meant for
production. Before launching to real users, configure a proper SMTP provider
so auth emails (signup confirmation, OTP codes) are reliable and come from a
professional address. Resend (resend.com) is the recommended option — free
tier covers small apps, easy setup, good deliverability. Requires a verified
domain. Gmail SMTP works as a stopgap without a domain.
Set up in: Supabase → Authentication → Emails → SMTP Settings.

--- Consider removing Vercel MCP plugin ---
The project deploys to GitHub Pages via GitHub Actions — Vercel is not used.
The Vercel MCP server shows a setup warning on every Claude Code launch because
it needs authentication. If we're confident we won't move to Vercel, remove the
plugin to eliminate the noise.

--- FEATURE: User-provided API keys ---
Allow users to paste their own Anthropic API key in a settings screen. The key
would be stored in localStorage (stays on-device, never sent to our servers
except as a request header to the Edge Function). The Edge Function would use
the user-supplied key instead of the server key when one is present. This
eliminates all API cost risk for the developer and lets power users run the app
without any per-use charges. Implementation notes:
  - Add a "Settings" screen or modal with an API key input field
  - Validate the key format client-side (starts with "sk-ant-")
  - Pass it as a custom header (e.g. X-User-Api-Key) to the Edge Function
  - Edge Function checks for the header and prefers it over the env-var key
  - Show a clear indicator in the UI when the user's own key is active
  - Consider only supporting Anthropic keys for now; other providers
    (OpenAI, Gemini) would require separate API request formats and
    system prompt tuning and are a larger project
